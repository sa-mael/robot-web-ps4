
Project Interceptor: Quadruped Robotics Platform (v75.0)OverviewProject Interceptor is an advanced, custom-built quadruped robot powered by an ESP32-S3 (Dual-Core, 8MB PSRAM) microcontroller. The firmware is built on a real-time operating system (FreeRTOS) architecture, featuring dual-core processing, raw non-blocking I2C sensor fusion, and complex 3D Inverse Kinematics (IK) for holonomic movement and active self-balancing.⚙️ Hardware ArchitectureTo ensure extreme stability and prevent signal collisions, the system utilizes a Dual I2C Bus topology, isolating the noisy servo PWM signals from the sensitive IMU telemetry.Microcontroller: Waveshare ESP32-S3-DevKit-N8R8Actuation (Bus 0): PCA9685 16-Channel PWM DriverSDA: GPIO 4 | SCL: GPIO 5 | FREQ: 400 kHzControls 16 Metal Gear Servos (4 Degrees of Freedom per leg: Coxa, Femur, Tibia, Twist).Sensor Fusion (Bus 1): MPU6050 6-DoF IMUSDA: GPIO 6 | SCL: GPIO 7 | FREQ: 100 kHzConfigured in RAW mode (register-level reading) with a 42Hz Digital Low Pass Filter (DLPF) and a complementary filter for stable Pitch/Roll tracking.Status Indication: On-board WS2812B RGB LED (GPIO 38)🔵 Booting | 🟢 Standby/Ready | 🟣 Walking | 🟠 Test/Diagnostics | 🔴 Hardware Error🧠 Software Architecture (FreeRTOS)The firmware utilizes FreeRTOS to pin critical tasks to specific silicon cores, guaranteeing that heavy network traffic never interrupts the 50Hz physical control loop. Data sharing across cores is strictly protected by a SemaphoreHandle_t Mutex.Core 0 (TaskNetwork):Handles Wi-Fi connectivity and serves the HTML/JS Flight Deck dashboard.Manages asynchronous WebSocket payloads (gamepad telemetry).Broadcasts 3D spatial data (Pitch, Roll, Leg Angles) back to the UI at 10Hz.Core 1 (TaskControl):Runs a rigid 50Hz (20ms) deterministic control loop (vTaskDelayUntil).Executes the Kinematics Engine, reads IMU data, and updates servo PWM pulses.📐 Kinematics & Locomotion EngineThe robot utilizes a custom-written Inverse Kinematics (IK) solver and Gait Scheduler.Creep Gait (3-Point Stability): Uses a strictly phased step sequence (FL -> BR -> FR -> BL). Three legs remain planted on the ground at all times, pushing the chassis forward, while one leg performs a sinusoidal swing trajectory.Holonomic Twist: The robot can yaw (twist) its body while moving linearly. The coordinate system dynamically rotates the target foot placements, keeping the feet "pinned" to the ground relative to the world frame.Active IMU Balancing: Regardless of gait state, the system continuously offsets the Z-height of each leg using $Z_{offset} = X \cdot \sin(Pitch) + Y \cdot \sin(Roll)$ to maintain chassis parallelism against uneven terrain.Kinematic Protections: * Features geometric hard-limits to prevent negative radius calculations ($L_{coxa}$ inversion) that could mechanically damage the chassis.Emplements Smooth Parking (velocity damping) to ensure legs land softly when joystick input ceases, preventing current spikes.💻 Control InterfaceThe firmware serves a completely custom "Flight Deck" Web UI accessible via any browser. It features a Three.js-powered 3D wireframe visualizer that mirrors the robot's physical joint angles and chassis orientation in real-time. It includes:Virtual Artificial Horizon (Pitch/Roll).Individual Joint diagnostics and offset tuning.Real-time Gamepad API integration (connect a PS4/Xbox controller to the browser to drive the robot).

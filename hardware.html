System Architecture

The Arttous platform is powered by the ESP32-S3 (N8R8) Dual-Core SoC running FreeRTOS (Firmware IK4=75.4). To prevent I2C starvation during high-speed calculations, peripherals are hardware-split across two independent I2C buses.

FreeRTOS Task Management

The standard single-threaded loop has been entirely deprecated. Processing is strictly locked to dedicated silicon cores to ensure the Kinematic Engine never misses a 50Hz deadline, preventing physical leg jitter.

CORE 0: TaskNetwork
Priority: 1 | Unpinned Timing

Handles asynchronous events: WiFi connection, HTTP Server rendering, and WebSockets JSON telemetry broadcast every 100ms. Non-blocking.

CORE 1: TaskControl
Priority: 2 | Strict 50Hz (20ms) Loop

Executes IMU sensor fusion, Inverse Kinematics (Law of Cosines), Smooth Amplitude Decay, and generates hardware PWM signals.

* Memory space between cores is secured via SemaphoreHandle_t stateMutex to prevent race conditions during payload parsing.

Kinematics API & Stabilization

Implementation of the 3DOF solver with active IMU balancing and NaN-protection clamps. Runs exclusively on Core 1.

// 1. Active Z-Axis Balancing (IMU Sensor Fusion) float p = constrain(pitch, -20.0f, 20.0f) * (PI_F/180.0f); float r = constrain(roll, -20.0f, 20.0f) * (PI_F/180.0f); balZ[i] = (lx * sin(p)) + (ly * sin(r)); // 2. Smooth Amplitude Decay (Soft leg parking) smoothVx += (targetVx - smoothVx) * (4.0f * dt); float lz = (currentHeight + gait.legZ[i]) - balZ[i]; // 3. Law of Cosines (NaN Protection Clamp prevents CPU crash) float D = sqrt(u*u + lz*lz); float cA = constrain((FEMUR_SQ + D*D - TIBIA_SQ) / (2.0f * FEMUR * D), -1.0f, 1.0f); float alpha = acos(cA);

Web UI & Digital Twin

The system features a zero-install, browser-based Mission Control interface served directly from the ESP32-S3's PROGMEM.

Telemetry & Control Architecture

  • Rendering: WebGL via Three.js for 60FPS 3D wireframe visualization.
  • Transport: Core 0 handles WebSockets (Port 81) broadcasting JSON telemetry at 100ms intervals.
  • Flight Deck: Real-time Pitch/Roll horizon, auto-gait toggles, and dual virtual joysticks linked via HTML5 Gamepad API.
  • Engineering: Direct access to the Kinematics API. Allows manipulation of individual axes (Coxa, Femur, Tibia, Twist).

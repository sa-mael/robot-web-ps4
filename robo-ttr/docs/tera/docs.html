Hardware Architecture

The Arttous platform is built around the ESP32-C6 SoC. Below is the strict pin mapping required for Rhodium v53 firmware compatibility.

Note: All Servo PWM lines must be routed through the PCA9685 driver via I2C. Do not connect servos directly to the ESP32.

I2C Bus Configuration

ESP32 PIN FUNCTION TARGET DEVICE
GPIO 6 SDA (Data) PCA9685 / MPU6050
GPIO 7 SCL (Clock) PCA9685 / MPU6050
GPIO 9 INT (Interrupt) MPU6050 Wake

Power Specification

The system operates on a split-rail power design. Logic voltage (3.3V) is isolated from Servo voltage (6V-7.4V) to prevent brownouts.

Kinematics API

The `KinematicsEngine` class handles all coordinate transformations. The solver expects coordinates relative to the shoulder joint.

Solve Inverse Kinematics

Calculates joint angles for a given target foot position (X, Y, Z).

void solve_IK(float x, float y, float z, int leg_id) { // 1. Calculate Coxa Angle float theta1 = atan2(y, x); // 2. Calculate Femur-Tibia distance float L = sqrt(x*x + y*y) - COXA_LENGTH; // 3. Apply Cosine Rule for Servo 2 & 3 float alpha = acos((FEMUR*FEMUR + D*D - TIBIA*TIBIA) / (2*FEMUR*D)); setServo(leg_id, theta1, alpha); }

Gait Scheduler

Controls the rhythmic timing of leg movements. Rhodium currently supports Trot, Crawl, and Bound gaits.

// Switch to "Trot" Gait robot.setGait("TROT", 250); // 250ms step duration
